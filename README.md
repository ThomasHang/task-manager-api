## controller / routes / middleware   这些结构分别代表什么意思?

| 文件夹          | 意义                | 类比生活             |
| ------------ | ----------------- | ---------------- |
| `routes`     | 路由层，定义“访问路径”和请求类型 | 门口的前台：告诉你去哪办事    |
| `controller` | 控制器层，写“真正要干的事”    | 各个功能科室：真正处理业务    |
| `middleware` | 中间件层，做“所有人都要过的安检” | 大厅的安检门：没带身份证就不让进 |
